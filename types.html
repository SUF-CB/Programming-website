Prodcedural Programming

Procedural programing are imperatives and functions that use many procedures that are combined within the main program. To make the program simpler, sub procedures are used which run as sequential control structures when the program is being run by the user. Procedural programming is very simple to understand as the actual program runs in a sequential/imperative way and it only requires a keyboard which means it is easy to use as it doesn’t have any events such as clicking.

Event Driven Programming

Event driven programs usually have a GUI interface because the controls used in the interface can be used as in Windows to trigger routines in the program code sub-procedures.
